## Turn 1

### 🤖 Assistant

针对Lab1构建一个支持约12条基础整数指令的RISC-V 5级流水线CPU，以下是为你规划的完整实现路径、限制要求以及对应的核心知识点：

### 第一阶段：架构设计与模块划分
在正式写代码前，需要先规划好流水线的骨架。建议直接将5级流水线拆分为不同的独立模块，而不是全部写在一个庞大的逻辑块中。

- **实现任务**：在仓库的 `vsrc/core.sv` 文件中，建立取指（IF）、译码（ID）、执行（EX）、访存（MEM）、写回（WB）五个基础模块，并设计段间寄存器（Pipeline Registers）用于传递状态。
- **课程知识点**：单周期处理器模型与流水线性能比较；流水线数据通路的设计（《6_指令流水线.pdf》第一讲）；时序逻辑与状态存储元件的作用。
- **工具提醒**：在你使用 Cursor 辅助搭建基础框架或了解项目结构（如运行 `skill.sh` 脚本）时，切忌直接采用大段未经验证的生成代码。大模型生成的段间时序逻辑极易出现打拍错误。

### 第二阶段：指令译码与执行
实现从指令内存抓取指令后，对其进行解析并完成对应的算术逻辑运算。

- **实现任务**：
  1. 根据低7位 `opcode` 判断指令类型（如I型）。
  2. 根据具体格式切分出 `rd`（目的寄存器）、`rs1/rs2`（源寄存器）、`funct3/funct7`（功能码）以及 `imm`（立即数）。
  3. 配置 ALU 完成加法、异或等具体操作。特别是对于带有 `W` 后缀的指令（如 `ADDIW`），需要将计算结果的高32位进行符号扩展或掩码截断，以兼容32位操作。
- **课程知识点**：RISC-V RV64I 指令集格式（《4_指令系统.pdf》）；组合逻辑控制单元设计与 ALU 数据通路实现（《5_中央处理器.pdf》）。
- **重要限制 (⚠️ 硬件红线)**：**Verilog 中严禁直接使用 * 或 / 运算符**。由于组合逻辑无法在单个时钟周期内完成64位乘除法（门延迟过大会导致严重的时序违例），如果想要做乘除法的扩展实验，必须将其设计为多周期的状态机操作。

### 第三阶段：解决数据冒险与流水线冲突 (Lab1 核心难点)
由于流水线的特性，后序指令可能会依赖前序指令尚未写回寄存器的数据，这是导致 CPU 运行错误的主要原因。

- **实现任务**：
  - **转发 (Forwarding)**：实现前递网络，将 EX 阶段或 MEM 阶段的计算结果直接旁路（Bypass）送回 ID 阶段的 ALU 输入端。
  - **阻塞 (Stall)**：对于 Load-Use 冒险（例如前一条是 `lw` 指令，下一条紧接着要用该数据），仅靠转发无法解决，必须插入气泡（Bubble）让流水线暂停一个时钟周期。
- **课程知识点**：数据冒险（RAW）的现象；转发检测与控制逻辑（Forwarding Unit）；阻塞检测机制（Hazard Detection Unit）（《6_指令流水线.pdf》第二讲）。
