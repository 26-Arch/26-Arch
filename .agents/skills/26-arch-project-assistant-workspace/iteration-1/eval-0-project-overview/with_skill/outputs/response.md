# 26-Arch 仓库快速理解

## 这个仓库整体在做什么

`26-Arch` 是面向课程设计的 **CPU 工程仓库**，而不是从零搭 RTL 的空白模板。仓库已经搭好了：

- **仿真与构建**：Verilator 路径（`verilate/`）、顶层仿真集成
- **参考模型联仿**：`difftest/`，用于与参考 ISA 行为逐周期对照
- **预编译/汇编测试**：`ready-to-run/`，按 Lab 分组
- **板级路径**：`vivado/` 等，供后续上板

你的主要工作是在既有工程结构里 **实现并迭代 CPU 核心 RTL**，用标准 `Makefile` 目标做回归，而不是自己发明一套仿真流程。

---

## Lab1 / Lab2 大致在推进什么能力

这是课程主线上的**能力边界**（来自项目参考说明；具体是否已通过某测试以 `status.md` 与实测为准）。

### Lab1

侧重：

- 基础整数指令的正确执行路径
- 取指 → 译码 → 执行 → 写回的完整数据通路（以及课程要求的流水线/阶段划分）
- 能在 Difftest 下看到 **架构状态与参考模型一致**

`lab1-extra` 通常比基础 `lab1` 更“探边界”（更多指令类别/更强组合），**通过基础 Lab1 不等于自动满足 extra**。

### Lab2

侧重：

- **访存**：load/store 行为、与总线/存储器接口的请求与响应
- **宽度与扩展**：按访存宽度选择、符号/零扩展等与内存数据相关的细节
- 与工程里提供的 **内存/总线路径** 更真实地打交道

### 与“当前实现快照”的关系（易变信息）

截至维护中的快照（见仓库内 `status.md`）：实现更接近 **基础 Lab1 子集** 的流水线框架；`test-lab1` 通过、`lab1-extra` 与 `lab2` 目标尚未验证为通过。读代码时请以 **源码与测试** 为准，不要把快照当成永久结论。

## 若要读“当前 CPU 实现”，建议先看哪些文件、为什么

下面顺序兼顾 **“从外到内”** 和 **“先接口后细节”**。

### 1. `Makefile`

**为什么先看**：这是本仓库的 **标准命令与目标入口**（如 `test-lab1`、`test-lab2`）。先知道怎么编、怎么跑、测什么，再读 RTL 不容易迷路。

### 2. `vsrc/SimTop.sv`

**为什么先看**：仿真顶层，把 **CPU 核心、总线、存储器、Difftest** 接在一起。你要理解“你的 CPU 在仿真里连了谁、时钟/复位/提交点在哪里”，这里是总览。

### 3. `vsrc/include/common.sv`

**为什么先看**：共享类型、常量、总线侧约定。后面读 `core` 时，译码结构体、握手、位宽等都会引用这里；先扫一遍能减少反复跳读。

### 4. `vsrc/src/core.sv`

**为什么先看**：**CPU 实现的主入口**。当前仓库把五段流水线式实现落在 `vsrc/src/`，`core.sv` 是把这些阶段、模块连起来的中心。

### 5. `vsrc/src/` 下的子模块（按你关心的路径选读）

当前目录里典型包括：`core_decode.sv`、`core_alu.sv`、`core_regfile.sv`、`core_forwarding_unit.sv`、`core_hazard_unit.sv` 等。

**为什么**：具体指令如何译码、ALU 做什么、寄存器堆与转发/阻塞如何配合，都在这些文件里；比只看 `core.sv` 一行行跟更清晰。

### 6. `ready-to-run/lab1/*.S`（需要理解“Lab1 到底测什么”时）

**为什么**：测试汇编是 **能力边界的实用参照**——指令组合、访存是否出现、控制流复杂度等，往往比抽象描述更直接。

若你已在做 Lab2 相关，可对照 `ready-to-run/lab2/*.S`。

### 补充：仿真 vs 上板

- **`SimTop.sv`**：仿真与联仿视角。
- **`VTop.sv` / `mycpu_top.sv` 与 `vivado/`**：综合/上板路径。

讨论行为时要分清自己看的是哪条路径，避免把仿真里的钩子默认当成板级行为。

## 小结一行

本仓库让你在 **现成仿真 + Difftest + 分 Lab 测试** 的框架里迭代 CPU；Lab1 先把整数通路与基本正确性立住，Lab2 再重点攻访存与总线侧交互；读实现请从 **`Makefile` → `SimTop.sv` → `common.sv` → `core.sv` → 各 `core_*.sv`** 这条线入手，并用 `ready-to-run/` 对齐实验要求。
