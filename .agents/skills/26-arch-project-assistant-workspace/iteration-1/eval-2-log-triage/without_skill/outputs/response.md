# `make test-lab2` 日志摘要

## 构建 vs 运行

- **构建**：成功。Verilator 生成与 C++ 链接均完成，产物为 `build/emu`，仅有编译告警（如 `remote_bitbang`、`goldenmem` 等），无致命错误。
- **运行**：失败。属于**仿真/功能**问题，不是链接或编译失败。

## 最关键的失败位置

- **中止点**：`Core 0: ABORT at pc = 0x80000014`（与 trace 中的 `0x0080000014` 同一地址）。
- **对比点**：`a7 different at pc = 0x0080000014, right= 0x0000000040010000, wrong = 0x0000000000000000`。
- **上下文**：在 `commit inst [5]` 处，指令为 `400108b7`（典型为 **LUI a7, 0x40010**），参考模型上 `a7` 应为 `0x40010000`，你的设计侧为 `0`。

## 最可疑的问题类型

- **ISA / 数据通路**：上立即数类指令（尤其是 **LUI**）的立即数拼接、写回目标寄存器或提交路径有误，导致 `a7` 未正确写入或与参考不一致。
- 次要提示：日志中有 `spike-dasm: not found`，一般**不**是本次 ABORT 的主因；`find: '../src/main/scala': No such file` 出现在构建阶段，未阻止生成 `emu`。

优先打开你工程中实现 **Lab2 CPU 核心与写回** 的顶层 RTL（本仓库常见为 **`vsrc/core.sv`** 或与之等价的流水线/执行模块），重点核对：**`lui`（及立即数到寄存器的数据通路）在 PC `0x80000014` 附近的行为**，以及该条指令是否真的在提交时把 `0x40010000` 写入 `a7`。
